typedef uint8_t dma_peripheral_trigger_t

SOC-agnostic DMA peripheral trigger identifier.

This type should be defined by SOC-specific headers to map to actual peripheral trigger identifiers (e.g., DMA_TRIGGER_UART0_RX, DMA_TRIGGER_I2C_M0_TX).

◆ dma_event_callback_t

typedef void(* dma_event_callback_t) (dma_instance_en instance, uint32_t channel, uint32_t event, void *user_data)

DMA event callback function type.

This callback function is invoked when DMA events occur on a channel. The callback is executed in interrupt context, so it should be kept short and avoid blocking operations.

Parameters
[in]instanceDMA controller instance identifier
[in]channelChannel number that generated the event
[in]eventEvent flags indicating what occurred (combination of dma_status_flags_en)
[in]user_dataUser-provided data pointer passed during channel configuration
Note
This function is called from interrupt context. Keep processing minimal.

◆ dma_descriptor_handle_t

typedef uint32_t* dma_descriptor_handle_t

Handle to a DMA descriptor.

Represents a pointer to a DMA descriptor in memory. This handle is used to configure, link, and manage DMA descriptor chains for advanced transfer operations.

Note
The descriptor must be allocated and aligned as per DMA hardware requirements (typically 32-byte alignment). The handle has to be assigned with a descriptor buffer.
